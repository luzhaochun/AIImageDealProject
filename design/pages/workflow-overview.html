ImageDeal 图片协作平台 — 业务全流程
13 步图片生产闭环 · CSV 导入 → 领图作图 → 一审/二审 → 终稿归档 · ImageDeal 项目产品说明
步骤 操作人 操作 系统行为 数据 / 状态变更 图库归属
1-导入 管理员 上传 CSV 文件 解析 CSV → 读取图片下载链接 → 批量下载原图 image_urls 写入 images 表 —
2-入库 系统 自动下载图片 逐一下载 → 校验格式/尺寸 → 生成缩略图 images 表插入 N 条记录 原始图库（待分配）
3-分类 系统 按 CSV 类别归类 CSV 的 category 字段写入图片元数据 images.category 填充 原始图库（按类别分好）
4-领图 用户 选类别 → 领一张图 从该类待分配图库取 1 张，分配全局唯一编号 status=已分配，assigned_to=用户，生成 global_no 原始图库 → 已领出
5-换图 用户 评估后认为不可用 当前图 → 废图堆；用户重新领图 原图 status=废弃，重新走步骤 4 已领出 → 废图堆
6-作图 用户 在线编辑图片 图层 / 裁剪 / 调色 / 文字等操作，随时保存 创建 image_versions + image_layers 工作中（用户任务区）
7-提审 用户 作图完成 → 提交一审 图片进入一审审核队列 status=一审待审 用户任务区 → 一审待审队列
8-一审 审核员 审核图片 对比原图审核 → 通过 / 驳回 / 打回 创建 review_records 记录 一审队列
9-一审通过 系统 审核通过 图片进入「一审通过图库」 status=一审通过 一审通过图库
10-一审驳回 审核员 审核不通过 退回给用户，附修改意见 status=驳回，用户重新编辑 退回用户任务区
11-二审 审核员 从一审通过库领图审核（可选） 从一审通过图库中抽取进行二审 status=二审待审 一审通过图库 → 二审队列
12-二审通过 系统 二审通过 进入「二审通过图库」（终稿库） status=二审通过 二审通过图库（终稿）
13-终稿归档 系统 终稿标记 完成的图片可导出 / 打包 status=已完成 终稿归档库
